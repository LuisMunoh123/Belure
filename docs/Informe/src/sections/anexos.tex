/* Eleccion del pivote */
int pivot_pos = 0;
switch (pivot_type) {
    case 1:
        pivot_pos = left;
        break;
    case 2:
        pivot_pos = right;
        break;
    case 3:
        pivot_pos = median_of_three(V, left, right, comp_f);
        break;
    case 4:
        pivot_pos = rand() % (right - left + 1) + left;
        break;
}

/* Mediana de 3 */
int median_of_three(Player V[], int left, int right, int (*comp_f)(Player *, Player *))
{
	int mid = (left + right) / 2;
	if (comp_f(&V[left], &V[mid]) > 0) { // a > b
		if (comp_f(&V[mid], &V[right]) > 0) { // a > b > c, mediana es b
			return mid;
		} else if (comp_f(&V[left], &V[right]) > 0) { // a > c >= b, mediana es c
			return right;
		} else { // c >= a > b, mediana es a
			return left;
		}
	} else { // a <= b
		if (comp_f(&V[left], &V[right]) > 0) { // b >= a > c, mediana es a
			return left;
		} else if (comp_f(&V[mid], &V[right]) > 0) { // b > c >= a, mediana es c
			return right;
		} else { // c >= b >= a, mediana es b
			return mid;
		}
	}
}

\subsection{Teorema Maestro}\label{sec:teorema-maestro}

El \textbf{Teorema Maestro} constituye una herramienta fundamental para determinar la complejidad temporal de algoritmos que siguen el paradigma de Divide y Vencerás, ya que proporciona un mecanismo sistemático para resolver ecuaciones de recurrencia de la forma:

\[
T(n) =
\begin{cases}
  c & \text{si } n \leq n_0 \\[4pt]
  aT(n/b) + f(n) & \text{si } n > n_0
\end{cases}
\]

donde $a \geq 1$, $b > 1$ y $f(n)$ es una función asintóticamente positiva. Los parámetros $a$ y $b$ representan, respectivamente, el número de subproblemas en que se divide el problema original y el factor por el cual se reduce el tamaño de cada subproblema; mientras que $f(n)$ captura el costo adicional del trabajo realizado fuera de las llamadas recursivas (por ejemplo, la división del problema y la combinación de resultados). Bajo estas condiciones, el teorema establece los tres casos siguientes:

\begin{enumerate}
  \item Si $f(n) = O\!\left(n^{\log_b a - \varepsilon}\right)$ para alguna constante $\varepsilon > 0$, el trabajo de combinación es asintóticamente menor que el costo de los subproblemas y, por tanto: \[T(n) = O\!\left(n^{\log_b a}\right)\]
  \item Si $f(n) = O\!\left(n^{\log_b a}\log^k n\right)$ para alguna constante $k \geq 0$, ambas contribuciones son del mismo orden y: \[T(n) = O\!\left(n^{\log_b a}\log^{k+1} n\right)\]
  \item Si $f(n) = \Omega\!\left(n^{\log_b a + \varepsilon}\right)$ para alguna constante $\varepsilon > 0$, y se cumple la condición de regularidad $af(n/b) \leq c\,f(n)$ para alguna constante $c < 1$ y todo $n$ suficientemente grande, el costo de combinación domina y: \[T(n) = O(f(n))\]
\end{enumerate}

Para los algoritmos analizados en este informe resulta conveniente emplear la \textbf{versión simplificada} del teorema, aplicable cuando $f(n)$ es una función polinómica de la forma $cn^k$:

\[
\boxed{
T(n) =
\begin{cases}
  c\,n^k & \text{si } 1 \leq n < b \\[4pt]
  a\,T\!\left(\dfrac{n}{b}\right) + c\,n^k & \text{si } n \geq b
\end{cases}
\;\implies\;
T(n) \in
\begin{cases}
  O(n^k)          & \text{si } \log_b a < k \\[4pt]
  O(n^k \log n)   & \text{si } \log_b a = k \\[4pt]
  O(n^{\log_b a}) & \text{si } \log_b a > k
\end{cases}
}
\]

En esencia, el resultado depende de la relación entre $\log_b a$ (exponente que refleja cuánto crece el árbol de recursión) y $k$ (exponente que caracteriza el costo por nivel): si el árbol de recursión domina, la complejidad queda determinada por su altura; si el costo por nivel domina, la complejidad viene dada por dicho costo; y si ambos están equilibrados, aparece un factor logarítmico adicional.
